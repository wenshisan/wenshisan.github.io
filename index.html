字符串

 字符串是有限的字符序列，理所当然，但当有人问什么是字符时真正的问题才得以出现。使用英语的人们所熟悉的常见标点符号和字母 A、B、C 等等都是字符。这些字符都被标准化地按照 ASCII 标准映射到 0 到 127 之间的整数数值。当然很多其他的字符确实在非英语语言中使用，包括多种多样口音和变型的 ASCII 字符，比如像是和西里尔文、希腊文有关联的文法，完全和 ASCII 和英语没有关联的文法，包含了阿拉伯语、中文、希伯来语、北印度语、日文和韩文。Unicode 标准处理了字符实际上是什么的复杂性，并且导致的这个问题通常作为明确的标准接受。取决于您的需求，您可以完全忽略这些复杂性，假装值存储在 ASCII 字符中，或者您可以编写当遇到操作非 ASCII 文本时任何可控的字符或编码。Julia 对 ASCII 文本的处理简单且高效，并且让简单高效地操作 Unicode 成为可能。特别是您可以编写 C 风格的字符串代码去处理 ASCII 字符串，他们也会全部都依照性能和语义一样预期的工作。如果一些代码遭遇到了非 ASCII 文本，它将会温和地抛出错误信息，而不是悄悄地产生出包含糟粕的结果。当这种情况发生时。编辑代码并去操作非 ASCII 数据也很简单直接。

整数和浮点数

 整数和浮点数是算法和计算的基础构件。尽管整数和浮点数直接值的表示在代码中被看作是数字字符，一些内置值的表示则被称之为数值类型。例如虽然 1.0 是一个浮点数字符， 1 是一个整数字符，他们在内存中的二进制对象都是数值类型。

 Julia 提供了一个范围广泛的基础数字类型并且在他们之上定义了标准数学函数还有完整的算法和按位运算的补集。这些直接在数字类型和运算上的映射都原生地被现代计算机所支持，因此让 Julia 可以最大限度的利用计算资源。此外 Julia 提供了可以控制无法有效地在原本的硬件中表示的数值计算的高精度计算的软件支持，但会导致较低性能。
