泛型程序是特性

来源于数学到泛型程序[SR14]

 泛型编程是一种专注于算法设计和数据结构的编程方法所以他们能在大多数不用降低效率的通用设置中工作，泛型编程是特性在前的编程而不是特定的一系列工具。

 在那个观念中，把泛型编程理解为一种可交互的方法去不用比对性能而不是作为一系列规则去揭示泛型

 就像我们将要观察到的，核心方法是把数据结构和算法看作是松耦合的，这是于面向对象编程继承方法直接对比。

 此文稿具有对泛型算法的介绍以及阐述 Julia 如何帮助泛型算法实现的双重角色。

浏览类型树

数据结构和操作他们的算法之间使用类型系统进行连接。

 具体类型（即 Float64 或 Array{Float64, 2}）都是我们应用算法的数据结构,并且抽象类型（比如相应的 Number 和 AbstractArray）在一系列关联的数据结构和算法之间提供了映射。

 除了 typeof 和 supertype 函数，一些其他对于分析类型树有用的工具都在类型简介文稿中进行讨论。

Using the subtypes function, we can write an algorithm to traverse the type tree below any time t – with the confidence that all types support subtypes
 使用 subtypes 函数，我们可以

from https://github.com/JuliaLang/julia/issues/24741

function subtypetree(t, level=1, indent=4)
if level == 1
println(t)
end
for s in subtypes(t)
println(join(fill(“ “, level indent)) string(s)) # print type
subtypetree(s, level+1, indent) # recursively print the next type, indenting
end
end
