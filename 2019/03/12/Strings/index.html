字符串

 字符串是有限的字符序列，理所当然，但当有人问什么是字符时真正的问题才得以出现。使用英语的人们所熟悉的常见标点符号和字母 A、B、C 等等都是字符。这些字符都被标准化地按照 ASCII 标准映射到 0 到 127 之间的整数数值。当然很多其他的字符确实在非英语语言中使用，包括多种多样口音和变型的 ASCII 字符，比如像是和西里尔文、希腊文有关联的文法，完全和 ASCII 和英语没有关联的文法，包含了阿拉伯语、中文、希伯来语、北印度语、日文和韩文。Unicode 标准处理了字符实际上是什么的复杂性，并且导致的这个问题通常作为明确的标准接受。取决于您的需求，您可以完全忽略这些复杂性，假装值存储在 ASCII 字符中，或者您可以编写当遇到操作非 ASCII 文本时任何可控的字符或编码。Julia 对 ASCII 文本的处理简单且高效，并且让简单高效地操作 Unicode 成为可能。特别是您可以编写 C 风格的字符串代码去处理 ASCII 字符串，他们也会全部都依照性能和语义一样预期的工作。如果一些代码遭遇到了非 ASCII 文本，它将会温和地抛出错误信息，而不是悄悄地产生出包含糟粕的结果。当这种情况发生时。编辑代码并去操作非 ASCII 数据也很简单直接。

这里有一些关于 Julia 字符串显著的高级特性：

  • 内置于 Julia 中用于字符串（还有字符文字）的固定类型是 String。这是通过 UTF-8 编码支持的全范围的 Unicode 字符。（提供了 transcode 函数去从/到其他 Unicode 编码转换）。
  • 所有的 string 类型都是抽象类型 AbstractString 的子类型，且扩展包定义了额外的 AbstractString 子类型（比如对其他编码）。如果您定义了一个期望以 string 作为参数的的函数，为了接受任意的 string 类型您需要把类型声明为 AbstractString 类型。
  • 类似于 C 和 Java， 但不同于多数动态语言，Julia 有一个称作 AbstractChar 用于表示单字符的优秀类型。内置的 AbstractChar 子类型 Char 是一个可表示任何 Unicode 字符的32位简单类型（以 UTF-8 编码为基础）。
  • 然而在 Java 中字符串是不可变的：AbstractString 对象的值是不能被改变的。去构造一个不同的字符串值您要从其他字符串的部件中构造。
  • 概念上字符串是从索引到字符的局部函数：对于一些索引值来说，并没有字符值返回，取而代之的却是异常的抛出。这允许在字符串中通过编码表示的字节索引进行高效的索引而不是使用字符索引，但这对于可变宽度的 Unicode 字符串编码来说不能同时实现高效性和简单化。
