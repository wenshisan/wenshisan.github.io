配置您的 Julia 环境

概览

 我们会在这个文稿中介绍如何建立和运行 Julia。 有几种不同的选项去使用 Julia，包括本地桌面安装和 web 端的 Jupyter 主机。如果您进入基于 web 的 Jupyter 和 Julia 配置，这是典型的最简单直接的开始方式了。

Jupyter 要点

 类似于 Python 和 R ，但不同于比如 Matlab 和 Stata 的产品，Julia 在作为一个编程语言和明确的开发环境之间是一个松散的联系。

尽管您最终会使用其他编辑器，但在学习 Julia 的过程中以 Jupyter 作为开始还是会有很多益处：

  • 混合格式化文本（包括数学表达式）和代码在单个文档中的能力。
  • 良好的格式输出包括表格、数字、动画、视频等。
  • 可以创建 PDF 幻灯片、静态 HTML 等的转换工具。
  • 无需安装的可用于在线的 Jupyter。
    我们将会在下个文稿中讨论这些特性上的流程。

在桌面上安装 Julia 和 Jupyter

如果您想在本地机器上安装这些工具，请您下载并安装二进制的 Anaconda Python https://www.anaconda.com/download/。

确认您您点击了 yes 去 “add Anaconda to my PATH”。

从下载页面下载并安装 Julia 接受所有默认选项

我们不推荐 JuliaPro 因为它包含了很多包

通过以下任一方式打开 Julia ：

  • 通过您的开始菜单或者桌面图标进入 Julia (Windows 或 Mac 系统)；
  • 打开终端输入 julia (Linux 系统)；
    您应该可以看到如下图一样的画面：

这个叫做 JULIA REPL（Read-Evaluate-Print-Loop 读取-求值-打印-循环），我们下面将会讨论更多。

在 Julia REPL 中键入 ] 进入扩展包模式然后输入：

add IJulia InstantiateFromURL

这个添加的包为了：

  • IJulia 核心连接 Julia 到 Jupyter (也就是说允许您的浏览器运行 Julia 代码，关联 Julia 扩展包等。)。
  • InstantiateFromURL 是一个由 QE 团队为文稿的依赖包管理而编写的工具。

安装 QuantEcon 扩展包

安装过程的下一个步骤是下载并安装 QuantEcon 文稿记录所用的所有扩展包。

接下来在主 REPL (在蓝色的扩展包模式键入回车键返回) 运行：

using InstantiateFromURL

接着执行以下（取决于您的网络连接和电脑这将会耗费 15-20 分钟时间）：

activate_github("QuantEcon/QuantEconLectureAllPackages", tag = "v0.9.6", add_default_environment = true)

这个函数将：

  • 下载两个文件，Project.toml 和 Manifest.toml ，包含了用于此文稿笔记扩展包的最新版本。
  • 安装这些扩展包到您的电脑。
  • 将他们添加到默认的 Julia 环境，让您可以在 QuantEcon 文稿之外也可以使用这些扩展包。

注意：
在 activate_github 中调用的 tag 对应的数字是 QuantEcon 文稿在特定时间点所使用的包的截图。

我们将会在以后更加深入地解释 “环境”。

启动 Jupyter

 下面让我们安装 QuantEcon 文稿笔记到我们的电脑并运行他们（对于我们将要使用的工具的更多细节，请查看我们文稿的版本控制）

1 安装 git
2 (可选，但是强烈推荐)安装 GitHub Desktop

GitHub Desktop 方法

注意：本流程需要 Git LFS(Large File Storage 大文件存储)

如果您之前没有用过 Git LFS, 选择在这些简介对话框后面显示的 “Initialize Git LFS”。

安装 Git Desktop 程序之后，在电脑上点击此连接自动安装 notebooks

它应该会像这样在 GitHub desktop 软件中打开一个窗口

选择一个您喜欢的路径并克隆仓库：

至此您应该可以看到 Git LFS 对话框：

点击接受

在顶部的 “Repsitory” 下拉菜单下方， 点击 “Open in Terminal” (MAC, Linux) 或者 “Open in Command Prompt” (Windows)。

注意：在 Windows中，您可能需要点击出现的 “Open without git” 按钮。

在最终对话结果中运行：

jupyter lab

您将可以打开 Interacting with Julia notebook 去浏览这个界面并开始编码

Git 命令行方法

如果您不想安装 GitHub Desktop，您可以使用 Git 命令行工具获取 notebooks:

首先，执行以下去安装 Git LFS（Linux/OSX）

从最后的命令输出上您应该看到 Git LFS 已经安装。

打开新终端对话并运行：

git clone https://github.com/quantecon/quantecon-notebooks-jl

这将会在工作目录下载包含 notebooks 的仓库

然后 cd 进入您的 Mac、Linux 或 Windows PowerShell 终端

cd quantecon-notebooks-jl

然后运行：

jupyter lab

 打开和 Julia 文稿的交互（在 JupyterLab 中的 notebooks 列表的 julia_environment.ipynb 文件）并继续。

在 Web 端使用 Julia

如果您使用在线 Julia 安装，这是最简单的开始方式。

 最终您将会为了使用比如 Atom/Juno 的工具和编译器而进行本地安装，但请不要让环境成为学习的阻碍。

在JupyterHub 中使用 Julia

如果您使用基于 web 解决方案的 Jupyter, 这将会是典型的直接选项。

  • 学生：向您的科系咨询资源是否可用。
  • 大学或工作组：发邮件至 contact@equantecon.org 获得配置共享的包含预编译扩展包的 JupyterHub 实例以供这些文稿笔记所用。
  • JuliaBox 严格控制允许的扩展包，且目前不支持 QuantEcon 文稿。
获取 Notebooks

您的第一步是在您的 JupyterHub 环境中获取一个 notebooks 的拷贝。

 尽管您可用从网站单独下载 notebooks ,最简单的方式是使用 Git 克隆仓库到您的 JupyterHub 环境。

  JupyterHub 的安装有很多不同克隆仓库的方法，您可用使用 notebooks 的仓库 url: https://github.com/QuantEcon/quantecon-notebooks-jl

安装扩展包

 在您已经有了一些可用的 notebooks 之后，像上面那样，这些文稿取决于函数（比如用于测绘、基准、统计的包）在 web 端并非是一同安装于每一个 Jupyter 安装的。

 如果您线上的 Jupyter 并没有预安装的 QuantEcon 包，您可用安装 InstantiateFromURL 包，这是一个 QE 团队为此文稿编写的用于管理依赖包的工具。

在线上的 Jupyter 笔记本中运行（通常使用 ）

] add InstantiateFromURL

 如果您线上的 Jupyter 环境没有预安装这个包，您 QuantEcon 笔记本的第一次运行将会花费 15-20 分钟。

在这步之后，打开已下载的 Interacting with Julia 笔记本开始编码。
如果在此安装步骤之后 QuantEcon 笔记本仍无法工作，您可用需要咨询 JupyterHub 管理员。
