数学运算和初等函数

发表于 2019-03-06 | 更新于 2019-03-08 | 分类于 Julia中文文档 ， 手册 ， 数学运算和初等函数

Julia 提供了一个贯穿它全部简单数字类型的完整基础算术和按位运算符，也提供了大量便利高效的标准数学函数。

向量化的“点”操作符

 对每个像是 ^ 的二元运算来说都有一个相对应的 “点” 运算 .^ 就是自动被定义为在数组中依次执行元素的 ^ 运算。比如 [1,2,3] ^ 3 是没有定义的，因为没有标准的数学含义去 “三次方” 一个（没有平方）的数组，但是 .^ 3 被定义为按元素（或 “向量的”）产生 [1^3,2^3,3^3] 的结果。类似于像是 ! 或 √ 的一元运算符，也有与之对应的应用到每个元素的操作符 .√ 。

 更加明确的说，a .^ b 被解析为 “点” 调用 (^).(a,b)，这执行了一个广播运算：它可以组合数组和标量，相同大小的数组（依元素次序执行运算）和即使不同形状的数组（比如组合行和列的向量去产生一个矩阵）。而且和所有的向量化 “点调用” 一样，这些 “点运算符” 都有相熔性。比如如果您为数组 A 计算 2 .* A.^2 .+ sin.(A) （或等价的 @. 2A^2 + sin(A)，使用宏 @.），它会在 A 上执行一个独立的循环，为 A 的每个元素都执行 2a^2 + sin(a)。特别是像是 f.(g.(x)) 叠加的点调用也都是相熔的，并且类似于 x .+ 3 .* x.^2 “相邻的” 操作符都和叠加的点调用 (+).(x, (*).(3, (^).(x, 2))) 相等。

 此外 “加点的” 类似于 a .+= b （或 @. a += b） 升级操作符都被解析为 a .= a .+ b，.= 是一个合适的熔合赋值运算（查看加点语法文档）。

 注意点的语法也同样适用于用户定义的操作符。比如您为克罗内克积（kron）把定义 ⊗(A,B) = kron(A,B) 给了一个方便的中缀语法 A ⊗ B，之后 [A,B] .⊗ [C,D] 将会在没有额外代码的情况下计算 [A⊗C, B⊗D] 。

 和数字字符组合的点运算符可能会有歧义。比如会不清楚 1.+x 的意思是 1. + x 或是 1 .+ x.。因此不允许这种语法，并且这些情况下必须使用在操作符旁加上空格。

链式比较

不像显著除去 Python 的很多语言，比较可以是任意的链式：

 链式比较常常在数字代码中相当地方便。链式比较对于标量比较使用 && 运算符，为按元素位的比较使用允许他们能工作在数组上 & 运算符。比如 0 .< A .< 1 给了在 A 中对应的元素在 0 和 1 之间的条目一个布尔数组。

注意链式比较的等于行为：

 中间的表达式只会求一次值，而不是两次，因为将会在如果此表达式写成 v(1) < v(2) && v(2) <= v(3) 中求值。然而链式比较中的顺序是没有定义的。这里强烈不推荐在链式比较中使用有其他作用的表达式（比如打印）。如果需要其他作用，需要显式地使用短路 && 运算符（查看短路求值）。

基本初等函数

 Julia 提供了大批的数学函数和运算符。这些数学运算定义在广泛的数值种类上以进行合理的规范，包括整数、浮点数、有理数和复数，在任何这种规范可以合理的地方。

 而且这些函数可以使用点语法 f.(A) 以“向量化”的形式应用到数组和其他集合中，例如 sin.(A) 将会计算数组 A 中每个元素的正弦。
